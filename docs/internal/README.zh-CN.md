# 内部文档说明

这个目录只服务维护者，不作为公开首页入口。

## 这次发布后的文档分层

### 对外文档

- 根 `README.md`
  - 英文首页
- `README.zh-CN.md`
  - 中文首页
- `docs/en/*`
  - 对外英文细节文档
- `docs/zh/*`
  - 对外中文细节文档

### 内部文档

- `docs/internal/*`
  - 只保留中文
- `docs/superpowers/*`
  - 规划、设计、实施过程文档

## 维护约定

1. 公开 README 先讲定位、动机、场景，再讲命令。
2. 面向技术实现或维护者的高细节内容，优先放到 `docs/en/*`、`docs/zh/*` 或内部文档，不直接堆在首页。
3. 公开文档避免出现本机用户名路径、私有主机信息和一次性排障细节。
4. 中文文案保持专业、简洁，必要时保留稳定英文术语。
5. 仓库对外定位是更通用的 session vault，但 Tokscale 是当前旗舰 use case，公开文档不能把这一点写丢。

## 发布前人工检查

每次准备对外发布时，至少做下面几项检查：

1. `README.md` 与 `README.zh-CN.md` 的结构和主张是否仍然一致
2. `docs/en/*` 与 `docs/zh/*` 是否跟当前 CLI 能力保持一致
3. 示例配置是否仍然通用，是否泄露了私有路径或机器信息
4. `pyproject.toml` 的元信息与安装说明是否匹配
5. 测试、CLI help、可编辑安装是否仍然通过

## 当前公开叙事边界

当前公开叙事重点是：

- 多机、多目录、多客户端的 session 管理
- projection-first 同步
- Tokscale raw / canonical 视图
- 归档与冷热分层

当前不应该在公开首页过度展开的内容：

- 一次性排障过程
- 私有机器同步细节
- 针对单次异常 session 的 forensic 过程
- 与某个 provider 账单口径不一致的细粒度调查记录
